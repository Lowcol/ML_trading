# Data Module

This folder contains data ingestion and feature-engineering scripts for the trading workflow.

## Files

### `api.py`

`api.py` downloads market data from Tiingo and writes CSV files used by downstream steps.

Main behavior:

1. Loads `TIINGO_API_TOKEN` from your environment (`.env` supported via `python-dotenv`).
2. Fetches daily adjusted data (`adjOpen`, `adjHigh`, `adjLow`, `adjClose`, `adjVolume`).
3. Fetches intraday IEX candles (`open`, `high`, `low`, `close`, `volume`).
4. Saves outputs to this folder:
   - `data/spy_history.csv`
   - `data/tsla_eod.csv`
   - `data/tsla_intraday.csv`
5. Converts TSLA EOD data to a NumPy tensor for ML-ready numeric input.

### `preprocessing.py`

`preprocessing.py` takes a Tiingo EOD CSV (like `data/tsla_eod.csv`) and builds breakout-oriented features.

What it computes:

1. Sorts rows by `date`.
2. Momentum features:
   - `mom_90d`: 90-day return
   - `mom_30d`: 30-day return
3. Trend features:
   - `ema10`, `ema20` on `adjClose`
4. Volatility/tightness features:
   - `daily_range = (adjHigh - adjLow) / adjLow`
   - `adr_20d`: 20-day average daily range
   - `vol_contraction`: 5-day range average divided by `adr_20d`
5. Volume confirmation features:
   - `avg_vol_20d`
   - `rel_vol = adjVolume / avg_vol_20d`
   - `vol_dryup`: 5-day average volume divided by `avg_vol_20d`
6. Market Context & Relative Strength:
   - `market_regime`: `1` if SPY > SPY 200 EMA, `0` otherwise
   - `spy_mom_90d`: SPY's 90-day return
   - `relative_strength_90d`: `mom_90d` - `spy_mom_90d`
7. Supervised labels for breakout candidates:
   - Candidate filter: `mom_90d > 0.30`, `vol_contraction < 0.75`, `vol_dryup < 0.80`, and **`market_regime == 1`**
   - Look-ahead window: next 10 trading days
   - Label `1` (success): +20% target is reached before -5% stop
   - Label `0` (failure): -5% stop is reached first
   - Non-candidates keep no label
8. Market Regime constraint:
   - Uses `SPY` data to compute `market_regime = (spy_close > spy_ema200)`.
   - Result is `1` for bull market and `0` for bear market.
9. Feature-only normalization for neural models:
   - Raw price columns (`adjOpen`, `adjHigh`, `adjLow`, `adjClose`) are **not** normalized.
   - Scale-invariant features are normalized safely without look-ahead (data leakage) using a 252-day **rolling** Z-score:
     - `mom_90d`, `mom_30d`
     - `relative_strength_90d`
     - `rel_vol`
     - `vol_contraction`
     - `vol_dryup`
     - `% distance from EMA` features: `pct_from_ema10`, `pct_from_ema20`
   - Normalized columns are written with `_norm` suffix (for example `mom_90d_norm`).
   - `market_regime` is specifically **excluded** from normalization and passed to the model as a raw `1` or `0` flag.

The function removes warm-up rows from rolling calculations and returns a model-ready DataFrame with feature, label, and normalized feature columns.

### `build_dataset.py`

`build_dataset.py` acts as the final aggregator for your machine learning training data.

1. Searches the `data/processed/` folder for every output generated by `preprocessing.py`.
2. Selects **only** the candidates (`is_candidate == True`) that resulted in a conclusive win or loss (`label.notna()`).
3. Appends a `ticker` column to each row to identify which stock the signal belongs to.
4. Concatenates all symbols into a single `master_training_data.csv`.
5. Prints the class distribution (Ratio of `1` vs `0`). If you have under 500 successful breakouts, it will warn you to process more history or symbols.

## Current CSV workflow

1. Run `python data/api.py` to create source CSV files in `data/raw/` (or acquire raw data through your own means).
2. Run `python data/preprocessing.py` to build breakout features for all files in `data/raw/` and save them to `data/processed/`.
3. Run `python data/build_dataset.py` to scrape all the "candidates" across all processed tickers and compile them into a single `master_training_data.csv`. This master file will contain purely labeled rows (1s and 0s) to be fed directly into your neural network classifier.

## Notes

- Failed API requests currently return empty lists/DataFrames.
- If token is missing, `api.py` raises a clear error.
